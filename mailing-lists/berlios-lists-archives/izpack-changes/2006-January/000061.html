I am investigating whether we can use IzPack  for
installing the application we've created.  My initial
progress  in configuring an installer was great, but
then I hit a snag with  the final parts of our
installation procedure.

The application to be installed requires a
configuration file of  its own, and this file must be
written at the time the application  is installed.  A
few of the pieces of information that go into the 
config file need to be queried of the person doing the
installation  (such as the sysadmin's name and email).
 Is there a way to write  out this configuration file
based on values entered in a user input  panel?

Also, the application requires either JRun or Tomcat to
be running  on the installation server.  I plan to ask
the person doing the  installation which of these is
present because there is a  particular file that we
don't want to install if they are using  JRun (because
JRun has its own version of this file).  Is there a 
way to make the installation of a particular file
conditional based  on user input?
