Thanks guys, for fixing the internationalization on my excludeGroups. I 
had to do some i18n work myself recently and found a few more problems 
with the izpack  - the column names of the InstallationGroupPanel, the 
names of the installation groups and the unit strings (MB, KB...) were 
not localizable. The code for localizing the unit strings is a bit ugly 
and I am thinking how to make it better, but this will work for now.

This is how the installation groups localization works for a sample 
group named "all":
   <str id="InstallationGroupPanel.groups.all" txt="translated 'all'"/>

If you don't like something, let me know.
