As indicated in your comment, here under the proposal.

The idea of the JBossPathPanel comes with our need to install applications for the JBoss application server.
It replaces the TargetPanel in our products setups.
It is in fact a "clone" of the JDKPathPanel and its usage is similar:
It creates its own output variable (if that variable was not set in the variable section) that can be used in other panels: JBossPath.
It also checks for provided min and/or max versions (through JBossPathPanel.minVersion and JBossPathPanel.maxVersion in the variable section).
Finally, it also takes care of the JBossPathPanel.skipIfValid when declared in the variable section.
Unlike the JDKPathPanel, it modifies the INSTALL_PATH variable, allowing its immediate usage (like in SummaryPanel or InstallPanel) just like the TargetPanel does, taking advantage of the appsubpath variable.

Some texts have been added to the langpack (sorry, we only add the texts for English and French languages), all containing "JBoss":
	JBossPathPanel.headline
	JBossPathPanel.extendedIntro
	JBossPathPanel.intro
	JBossPathPanel.info
	JBossPathPanel.badVersion1
	JBossPathPanel.badVersion2
	JBossPathPanel.badVersion3
	JBossPathPanel.summaryCaption

The build.xml file has been updated to include the new panel.
