Julien,

this might not be a bas idea. If we do this, one thing to think about is,
that not everyone will want to work with the newest version of IzPack.
Accordingly a number of older versions of the documentation should be
available.

In addition to browsing on line, there should be a download version of the
documentation as well.

available for on-line browsing only. While this is not bad, I would still
appreciate the ability to download the documentation.

Hi everybody,

In an attempt at making the documentation easier to maintain, I
suggest that we perform the following after 3.10.2 (no release date,
but I will propose one soon anyway).

1. Move it to a subsection of the IzPack website (which is actually a wiki).
2. Do not let this space be freely editable like the wiki space.
Instead have a set of volunteers with write access on this part of the
website.
3. Drop the documentation shipped with IzPack from 3.11.0.

The expected benefits are of course to make it easier for everyone to
improve the documentation and maintain it, but I guess that most
people will like to browser it directly from the website. Also, it may
help for people that come across IzPack for the first time as they can
have a look at how it works.

What do you think?
