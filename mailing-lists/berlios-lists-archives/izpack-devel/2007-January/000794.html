Not at all! I can imagine this can be true! What about my friend 
is not so familiar with computers and he/she asks me to install 
the software. He/she prepares all before it; even starts the 
installation. And then comes my turn, but language is "strange". 
  I can imagine situation when consultant comes to customer and 
starts installation with the language which consultant can't 
understand. Well, in this situation clever consultant change 
regional settings. Or if he/she is not so clever waits for new 
version of software where language can be selected or even only 
English version. All these don't make user happy. Of course I 
haven't give here any thought for the language in the software to 
be installed.

 > What we have to do here else, is to build an installer for each
 > language
 > (german & english) to ommit the language selection, which would
 > be
 > pretty curious...

No, just continue this procedure so making installers to _all_ 
those language. And the installed software might be only German 
or English (meaning installer's language is different as 
installed software). Or for others it would be English? Actually 
making separate installers has been true sometimes. But your 
suggestion would be more elegant.

I might think to use this feature but I prefer guideline: "keep 
it simple". This switch breaks only a bit the logical usage of 
the software. Using this switch won't disturb one person, but 
person who travels around to install software into computers 
different regional settings configured might think that there is 
some odd in this installer: it seems to work differently in some 
cases... But RTFM will help here... :-)
