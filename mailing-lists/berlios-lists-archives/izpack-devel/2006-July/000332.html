Hi,

I'd like to contribute what I needed for my project; hoping it is useful
for other people, too.

Scenario: The user has to select a file path during setup (using
SearchField), but then the selected file path is written as a property
into a Java Properties file. In there, backslashes need to be escaped.

So I needed a processor (to escape my backslashes) for SearchField.

The attached patch allows and handles the <processor class="xxx" />' tag
within the '<field type="search" />' tag.
