Dear colleagues,
all IzPack developer know that there are coding rules.
But sometimes we work at more than one project and they
differ in the coding rules. In the hurry it is possible
to use a wrong coding rule. The problem is, that if 
a file will be checked in with massive changes nobody
see what changed really.
Therefore please call
svn diff
before
svn ci
and look to the output. And if there are coding rule
changes in the diff please look whether they are right
or not.
In the moment I have checked in a little change in the
rules for eclipse to hold the layout of the header 
(license).

Thanks

Klaus
