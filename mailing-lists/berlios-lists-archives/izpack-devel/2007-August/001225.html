Marc & Vladimir,

From your mails I understood that there was a "createForAll" shortcutspec attribute that should be set to "true" in order to have a shortcut created for all users (if, of course, shortcut creation for all users is also selected at install time). That was the lack in my shortcutspecs.

I hadn't found the createForAll attribute in the docu and missed it (I just ran a "grep" on my whole izpack 3.10.2 docu dir : the only place where createForAll is found is in the javadoc, and I had only read the "user" xhtml doc).

It seems that I still have the problem with shortcuts on the (common) desktop, but at least now I get those in the menu. I will do some more testing and if I need to patch something I'll let you know.

Sorry for disturbing you.

Julien : I remember that the documentation effort was paused until you finished its refactoring to a new format. Is the new documentation format ready yet ? Can I update a file somewhere in the svn with some information about "createForAll" in the shortcutspec so that other people having the same problem will find the solution easier ?

Thanks all,

Loïc
