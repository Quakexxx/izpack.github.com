Hi,

It is already possible to define your own langpack-strings in a custom
langpack which is defined by a customlangpack-resource. I find it useful
if one can do the same for icons which are now defined in icons.xml. I
added a method loadCustomIcons to InstallerFrame which is called just
after loadIcons. It then looks for a resource called customicons.xml
which should have the same format as icons.xml. 

Attached you'll find a patch to integrate this into to izpack. Would be
cool, if this could go in the next izPack-version.

Regards,
