> you can change the contents of any variable in a custom action or custom
> panel or with <variable> in the installation description file (often named
> install.xml) because variables are not write protected and built-in
> variables are resolved at starting installation before resolving values
> given by <variable>.
> Build-in variables are created to get special information about the
> current system. If you override this, these information are away.
