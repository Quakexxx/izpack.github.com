I was looking on the web and could not find a list of Windows environment 
variables (I did once, but forgot to bookmark it -- iirc, it had a chart 
showing which variables are available in which versions).  I found it hard to 
find one, but several said typing "SET" and <ENTER> will give you a list of 
supported environment variables.

I'm not clear why this would be a problem.  While I have never gone into and 
examined the code in the VariableSubstitutor, if it works on Win2k or XP, it 
should work on Win98.

Here's a possible idea: Use the version of your install.xml that specifies the 
directory.  Make up a dummy batch file and use the same env variable in there 
and specify for it to be parsed.  See if it is parsed in the .bat file on 98 
and on the other versions.  I don't know how much it'll help knowing that, 
but it may help narrow down what is going on.
