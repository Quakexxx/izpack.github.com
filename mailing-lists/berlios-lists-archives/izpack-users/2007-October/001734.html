Actually problem is not with IzPack. Problem may be in ur shortcutSpec.xml file.
Check ur second shortcut target property again. if it is any error in
that path location izpack will take the first one's target location.
    or try to change the order of <shortcut> tag in shortcutSpec.xml.
it means put ur second <shortcut> tag at first(erroneous one). now u
can see the err msg when u click the shortcut if the location is not
correct.
