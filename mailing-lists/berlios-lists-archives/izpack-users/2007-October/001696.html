Hello,

On Mac OS X when specifying the '~' character as the first character in a
file selection dialog the '~' character is expanded to the user's home
directory, just like this is done by a Unix shell.
IzPack also seems to somehow expand '~', but I am now sure how. As far
as I can tell if '~' is the first character when entering the
installation path it is expanded to the current directory following by
'/~'.
Could somebody please explain IzPack's rule for this kind of expansion?
If this rule differs from what one finds on Unix-compatible systems, could
this rule be changed to be compatible with the standard expansion rule?
